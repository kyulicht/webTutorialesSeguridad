Tutorial de Hacking Ético: Hydra y SQLMap

Introducción a Hydra

Hydra es una herramienta de fuerza bruta usada para probar la seguridad de sistemas mediante ataques de fuerza bruta a servicios de autenticación. Es capaz de atacar diferentes tipos de servicios como FTP, SSH, Telnet, HTTP, HTTPS, entre otros.

Comandos Básicos de Hydra

Para usar Hydra, la estructura básica del comando es la siguiente:

hydra [opciones] [tipo_de_servicio]://[dirección_del_objetivo]

Algunos comandos y opciones importantes:

  • -l o -L: Especifica un nombre de usuario único (-l) o una lista de usuarios (-L).
  • -p o -P: Especifica una contraseña única (-p) o una lista de contraseñas (-P).
  • -t: Define el número de tareas paralelas (usualmente se usa para acelerar el proceso).
  • -s: Especifica el puerto a atacar si es diferente al puerto estándar.

Ejemplo básico:

hydra -l admin -P /path/to/passwords.txt ftp://192.168.1.10

En este ejemplo, Hydra intentará acceder al servicio FTP en la dirección 192.168.1.10 usando el nombre de usuario admin y todas las contraseñas del archivo passwords.txt.

Introducción a SQLMap

SQLMap es una herramienta que automatiza la detección y explotación de vulnerabilidades de inyección SQL en aplicaciones web. Es una herramienta crucial para realizar pruebas de penetración y evaluaciones de seguridad.

Comandos Básicos de SQLMap

SQLMap tiene una estructura básica de comandos como la siguiente:

sqlmap -u [url_del_objetivo] [opciones]

Algunos comandos y opciones importantes:

  • -u: Especifica la URL del objetivo que se va a escanear.
  • --dbs: Muestra las bases de datos disponibles en el objetivo.
  • --tables: Muestra las tablas de una base de datos seleccionada.
  • --dump: Extrae los datos de una tabla específica.

Ejemplo básico:

sqlmap -u "http://example.com/vulnerable.php?id=1" --dbs

Este comando examina la URL proporcionada para ver si es vulnerable a inyección SQL y, si lo es, enumera las bases de datos disponibles.

Situaciones de Uso de SQLMap

SQLMap se utiliza cuando se sospecha que una aplicación web puede ser vulnerable a inyección SQL, lo que podría permitir que un atacante acceda o manipule bases de datos. Es especialmente útil en pruebas de penetración para identificar y explotar estas vulnerabilidades de manera controlada.
