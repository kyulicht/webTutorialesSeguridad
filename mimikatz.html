Tutorial de Mimikatz

Introducción

Mimikatz es una herramienta de código abierto utilizada para la extracción de credenciales en sistemas Windows. Es especialmente útil en auditorías de seguridad y pruebas de penetración. A continuación, exploraremos los comandos más comunes de Mimikatz y en qué situaciones se utilizan.

Comandos básicos de Mimikatz

1. Ejecutar Mimikatz con privilegios elevados

Antes de usar Mimikatz, es necesario ejecutarlo con privilegios de administrador. Esto se hace normalmente desde una ventana de comandos elevada:

mimikatz.exe

2. Mostrar credenciales en texto plano

Este comando muestra las credenciales almacenadas en la memoria en texto plano:

privilege::debug sekurlsa::logonpasswords

3. Exportar el archivo SAM y SYSTEM

Estos comandos permiten extraer las bases de datos SAM y SYSTEM para realizar un análisis posterior:

token::elevate lsadump::sam

4. Recuperar hashes de contraseñas NTLM

Este comando recupera los hashes NTLM de los usuarios del sistema:

lsadump::lsa /inject /name:krbtgt

5. Dump de tickets Kerberos

Con este comando puedes volcar los tickets Kerberos activos en el sistema:

sekurlsa::tickets

6. Ejecutar Mimikatz desde PowerShell

Es posible ejecutar Mimikatz directamente desde PowerShell con el siguiente comando:

Invoke-Mimikatz -Command 'sekurlsa::logonpasswords'

Guía de uso de Mimikatz

Mimikatz es una herramienta poderosa y debe utilizarse con responsabilidad. Aquí se presentan algunos escenarios comunes en los que se podría utilizar:

1. Auditoría de seguridad

Un auditor de seguridad podría usar Mimikatz para verificar si las credenciales están adecuadamente protegidas en el sistema.

2. Pruebas de penetración

Durante una prueba de penetración, un pentester podría usar Mimikatz para obtener acceso adicional dentro de un sistema comprometido.

3. Análisis forense

En un escenario de análisis forense, Mimikatz puede ser útil para comprender cómo un atacante podría haber accedido a información sensible.
